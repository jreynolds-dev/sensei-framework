# My Express API - Concept Cheat Sheet

Generated by `/pizza-time`. One-line summaries of every concept covered so far.

---

## Node.js Fundamentals

- **require vs import** — `require` is CommonJS (synchronous, dynamic); `import` is ESM (static, tree-shakeable). Choose based on your package type.
- **async/await** — syntactic sugar over Promises; `await` pauses execution inside an `async` function and unwraps the resolved value or throws on rejection.

## Express Routing

- **app.use() vs app.get()** — `use` matches any method on the path prefix; `get` matches only GET requests on the exact path.
- **Route parameters** — `:id` in a route pattern becomes `req.params.id`; use for resource identifiers.
- **Query strings** — `?key=value` in the URL becomes `req.query.key`; use for filters and options, not resource identity.
- **Express app setup** — `express()` creates an app; `app.listen(port)` binds it; always separate app creation from listen for testability.

## Middleware

- **Custom middleware** — a function with signature `(req, res, next)`; call `next()` to pass control, `next(err)` to skip to error handling.
- **Error-handling middleware** — must have 4 arguments `(err, req, res, next)`; placed after all routes; Express routes to it when `next(err)` is called.

## Authentication / JWT

- **JWT structure** — three base64url-encoded segments: header (algorithm), payload (claims), signature; never put secrets in the payload.

## Database / Prisma

- **Prisma schema** — defines models (tables), fields (columns), and relations; `prisma generate` regenerates the client after changes.
